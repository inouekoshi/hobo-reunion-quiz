# 要求定義書： クイズ大会システム

## 1. 目的と背景
* **目的**: 卒業後5年の節目に、恩師と教え子が一体となって楽しめるチーム対抗のクイズ大会を実施する。
* **背景**: イベントの特別企画として、単なるクイズではなく「速度・度胸・共感」を競う独自のシステムを用いて、会場全体の熱量と一体感を最大化させたい。

## 2. 対象ユーザーと利用環境
* **一般参加者（プレーヤー）**:
  * 卒業生および恩師。全8チーム（1チーム約7名で構成。恩師もチームの一員として参加）。
  * 各チームに1台の端末（スマートフォンまたはタブレット）を配布し、チーム全員で相談しながら代表者が操作する。
* **司会・進行（管理者）**:
  * イベントの進行役（井上様などを想定）。
  * 手元の端末（PCやタブレット）から、クイズの進行、問題の切り替え、BGM演出などのコントロールを一括で行う。
* **観客・全体（プロジェクター投影）**:
  * 会場のスクリーンにPCからプロジェクターで画面を投影。
  * リアルタイムのランキングや解答の集計結果（マジョリティ投票のグラフ等）を全体に共有する。

## 3. 業務要件（イベント当日のフロー）
1. **導入・チーム分け**: 各チームに1台ずつ操作用端末（スマホ等）を配布し、ルールの説明を行う。
2. **システムアクセス・待機**: 配布された端末から自作のクイズシステムにアクセスし、各チームごとにログインして待機状態となる。
3. **クイズ進行（約10〜12問）**: 
   * 管理者が問題をスクリーンに表示し、同時に参加者端末へ問題を配信・解答受付を開始する。
   * チームごとに相談し、解答およびベッティング（自信度に基づいた x1/x2/x3 の倍率）を選択して送信する。
   * 締め切り後、正解発表ならびにリアルタイムランキングの変動を投影する。
   * （マジョリティ問題の場合は、全体集計結果・得票率をグラフ等で投影）
4. **結果発表・表彰**: 最終累計ポイントに基づく総合優勝・準優勝、および特殊なログデータ（不屈の精神賞、ギャンブラー賞など）に基づく特別賞を発表・表彰する。
5. **事後共有**: 大会終了後、各チームの詳細な戦績データが閲覧できるQRコードを全体に表示・配布し、二次会等の話題を提供する。トータルの所要時間は約35〜40分とする。

## 4. 機能への要求（ユーザーが実現したいこと）
### 参加者機能
* チーム単位で指定のURLから簡単にアクセスし、システムに参加できること。
* 管理者の進行に合わせて出題された問題と選択肢を端末で確認できること。
* 解答にあたり、ベッティング（x1, x2, x3）を選択し、解答を送信できること。
* 早押し要素があるため、送信ボタンなどの押下レスポンスが良いこと。

### 管理者機能
* 全8チームのシステム接続・待機状態を一覧で把握できること。
* 問題の出題、解答受付スタート、強制締め切り、正解発表、次の問題への遷移などの進行状態を完全に手元でコントロールできること。
* タイミングに合わせてBGMの操作などの演出を管理画面から行えること。
* リアルタイムな順位付けと、現在のランキングをスクリーン用に投影・切り替えができること。
* マジョリティ問題の集計を自動で即座に行い、グラフ等でスクリーンに投影できること。
* 所定の基準（ベット成功率、解答速度、マジョリティ一致率など）に基づき、特別賞の選定に必要なデータ・ログを抽出できること。

## 5. 制約事項・非機能要求
* **動作環境**: 会場のWi-Fi環境、または各端末のモバイル回線にて安定して動作すること。
* **操作性**: ITに不慣れな参加者でも迷わない、直感的でシンプルなスマホ・タブレット向けUIであること。
* **時間的制約**: 全体で35〜40分以内にスムーズに完結させるため、画面の遷移待ちやローディングの遅延を極力排除し、テンポの良いシステム動作を実現すること。

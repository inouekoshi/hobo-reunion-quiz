# 要件定義書： クイズ大会システム

## 1. システム構成・アーキテクチャ
* **フロントエンド**: Next.js (React) / Tailwind CSS
* **バックエンド**: FastAPI (Python) / WebSockets
* **データベース**: PostgreSQL / Prisma
* **リアルタイム通信**: WebSocketsによる全画面完全同期

## 2. 機能要件

### 2.1 参加者画面
* **ログイン**: チーム名と管理者発行のパスコードによる入室。
* **クイズ解答**:
  * **2段階解答フロー**: 「倍率選択」→「解答選択」の順で行う。
  * **ベッティング制約**: x3は1回、x2は2回まで。x1は無制限。
* **リアルタイムフィードバック**: 正解発表時、個別の結果と最新スコアを表示。

### 2.2 管理者画面
* **ルーム管理**: 大会セッション（Room）の作成とパスコード発行。
* **問題管理**: リアルタイムでの問題登録（通常・マジョリティ）。
* **進行コントロール**: 出題、締め切り、結果発表、最終結果発表のフェーズ遷移。
* **チーム監視**: 参加チームのリアルタイム入室・解答状況の把握。

### 2.3 投影画面（プロジェクター）
* **ライブ・リーダーボード**: 全チームの順位を常に可視化。
* **演出アニメーション**: 
  * 出題時の問題表示
  * カウントダウン演出
  * 順位変動（カウントアップ・順位入れ替え）アニメーション

### 2.4 クイズ・ポイント計算
* **通常クイズ**: `(基本点 100 + 速度ボーナス 0~50) × 倍率`。不正解はマイナス。
* **マジョリティ**: `(基本点 100 × 倍率) × 得票率(%)`。全員が正解扱いだが、多数派ほど高得点。

## 3. 非機能要件
* **リアルタイム性**: 管理者の操作から全端末反映まで 300ms 以内を目標。
* **障害耐性**: ページリロード時に直前の進行状態、および個別のベット残り回数を復元。
* **UX/Aesthetics**: プレミアムで動きのある、イベントの場を盛り上げるリッチなデザイン。

# 技術仕様書

本ドキュメントでは、「Party Quiz」で使用しているプログラミング言語、フレームワーク、ライブラリ、およびその選定理由等の技術要件を定義します。

## 1. フロントエンド (Frontend)

| 技術 / ツール | バージョン | 用途・役割 | 選定理由 |
| :--- | :--- | :--- | :--- |
| **Next.js** | 14.x / 15.x | フレームワーク (App Router) | ルーティングの容易さと、Reactエコシステムの最新機能をフル活用できるため。 |
| **React** | 18+ | UIライブラリ | コンポーネント指向の開発、Hooks(`useEffect`, `useState`)を用いたWebSocket状態管理のため。 |
| **TypeScript** | 5.x | 開発言語 | 静的型付けにより、APIの戻り値やReactプロパティの安全性を担保するため。 |
| **Tailwind CSS** | 3.x | スタイリング | ユーティリティクラス駆動で、プロジェクター向けのアニメーションや装飾を高速に実装するため。 |
| **PostCSS / Autoprefixer**| - | CSSビルドツール | Tailwind CSS の基盤として利用。 |

## 2. バックエンド (Backend)

| 技術 / ツール | バージョン | 用途・役割 | 選定理由 |
| :--- | :--- | :--- | :--- |
| **Python** | 3.9+ | 開発言語 | データ処理の記述しやすさ、AI/スクリプト等との親和性。 |
| **FastAPI** | 最新 | Webフレームワーク | 高速な実行速度、WebSocketのネイティブサポート、Pydanticによる型バリデーション。 |
| **Uvicorn** | 最新 | ASGIサーバー | FastAPIを非同期で動作させるための軽量なWebサーバー標準実装。 |
| **Pydantic** | 2.x | データバリデーション | APIのリクエスト/レスポンスボディの型定義とバリデーションを自動化。 |
| **Prisma Client Python**| 最新 | ORM | TypeScript向けのPrismaと同等のスキーマ定義で、非同期(asyncio)の型安全なDBアクセスが可能なため。 |

## 3. インフラ・ミドルウェア (Infrastructure)

| 技術 / ツール | バージョン | 用途・役割 | 選定理由 |
| :--- | :--- | :--- | :--- |
| **PostgreSQL** | 15.x | リレーショナルDB | 堅牢性、Prismaとの親和性。JSON処理なども将来的に拡張が容易。 |
| **Docker** | 最新 | コンテナ環境 | 開発環境のPostgreSQL立ち上げを統一・簡略化するため。 |
| **Node.js / npm** | 18+ | FW実行環境 | Next.js の基本要件。 |

## 4. コード品質とディレクトリ等の規約

- **エラーハンドリング**:
  - バックエンド：`fastapi.HTTPException`を使用し、フロントへ明確なステータスコードを返す。
  - フロントエンド：`try/catch` でAPIエラーを補足し、UI上に反映（エラーテキストのStateを更新）。
- **WebSocket実装**:
  - `src/hooks/useWebSocket.ts` を共通フックとして切り出し、接続・切断・メッセージの受信ロジックをカプセル化している。
- **データ永続化（フロント）**:
  - チームID、チーム名などのセッション情報は `localStorage` に保存し、ブラウザリロード時にも参加状態を維持する。

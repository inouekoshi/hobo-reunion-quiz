# デプロイ手順書 (Vercel + Render + Supabase)

本番環境（インターネット上）にクイズシステムを公開するためのデプロイ手順です。

## 🌐 全体構成
- **フロントエンド**: Next.js (デプロイ先: Vercel)
- **バックエンド**: FastAPI + Prisma (デプロイ先: Render)
- **データベース**: PostgreSQL (提供元: Supabase)

---

## 1. データベース設定 (Supabase)
1. [Supabase](https://supabase.com/) で新しいプロジェクトを作成します。
2. 左メニューの `Project Settings`（歯車アイコン） > `Database` にある **Connection string (URI)** を取得します。
   - 例: `postgresql://postgres.[ProjectID]:[Password]@aws-0-[Region].pooler.supabase.com:5432/postgres`
3. このURL文字列をコピーしておき、バックエンドの環境変数として使用します。

---

## 2. バックエンドのデプロイ (Render)
バックエンドは Python + FastAPI + Prisma を使用しています。Prismaエンジンのバイナリ解決に特別な設定が必要なため、以下の手順に従ってください。

### 2-1. Render サービス設定
1. [Render](https://render.com/) のダッシュボードから **New Web Service** を作成し、GitHubリポジトリ（`party-quiz`）を連携します。
2. 以下の基本設定を入力します：
   - **Root Directory**: `backend`
   - **Environment**: `Python 3`
   - **Build Command**: `./build.sh`
   - **Start Command**: `uvicorn app.main:app --host 0.0.0.0 --port $PORT`

### 2-2. 環境変数 (Environment Variables)
デプロイ設定画面で、以下の環境変数を追加します。
- `DATABASE_URL`: （手順1で取得したSupabaseのURL文字列）
- `PYTHON_VERSION`: `3.11.9`

### 2-3. デプロイとPrismaバイナリの注意事項
Render環境（Debian OpenSSL 3.0.x）ではPrismaバイナリの自動解決に問題（`BinaryNotFoundError`）が発生しやすい傾向があります。
そのため、本プロジェクトでは `backend/build.sh` 内でPrismaエンジンバイナリを直接 `curl` でダウンロードし、カレントディレクトリに強制配置するスクリプトを導入しています。
- **デプロイ方法**: 万が一ビルドエラーが発生した場合は、ダッシュボードから `Manual Deploy` > `Clear build cache & deploy` を実行し、キャッシュを完全にクリアした上で再ビルドしてください。

---

## 3. フロントエンドのデプロイ (Vercel)
フロントエンドは Next.js を使用しており、Vercelと非常に相性が良いです。

### 3-1. Vercel プロジェクト作成
1. [Vercel](https://vercel.com/) のダッシュボードから **Add New Project** を開き、GitHubリポジトリを連携します。
2. 以下の設定を入力します：
   - **Root Directory**: **`frontend` を選択します。** （ここを間違えるとビルドに失敗します）
   - **Framework Preset**: `Next.js` が自動選択されるはずです。

### 3-2. 環境変数 (Environment Variables)
デプロイする前に、Vercelの環境変数に以下の2つを追加します。
- `NEXT_PUBLIC_API_URL`: `https://[Renderで発行されたバックエンドURL]/api`
- `NEXT_PUBLIC_WS_URL`: `wss://[Renderで発行されたバックエンドURL]/ws`
  - *(※ WebSocket通信のため、`https` ではなく `wss` となる点に注意してください)*

### 3-3. デプロイの実行
- 環境変数を設定した後、「Deploy」ボタンを押します。
- ビルドが成功すると、参加者や管理者が実際にアクセスできる公開URL（例: `https://party-quiz.vercel.app`）が発行されます。
- 以降はこの公開URLを使用して、クイズ大会を実施します。
